Unity 2022+ / Tuanjie 2022+ / WebGL minigame first

面向 WebGL 和微信小游戏的 C# hot update runtime。

hotc233-unity 以 Unity 2022+ 为最低支持版本，固定安装在 Assets/neko233/hotc233-unity，保留 HybridCLR 常用接入习惯，并把性能验证放到真实 IL2CPP Player：热更 C# 经 hotc 转换后运行，再与原生 Mono、原生 IL2CPP 和 HybridCLR 专业版纯解释目标区间比较。

优势

低内存优先

面向 WebGL/minigame，默认使用 RuntimeFast、原始 .dll.bytes payload、metadata 完整性校验、低 GC 热路径和 retained heap 哨兵。

HybridCLR 习惯兼容

保留 `HybridCLR.RuntimeApi` facade、兼容菜单别名和配置迁移入口，打包生态可用 class name 替换接入。

自动化性能闭环

`hotc233ctl headless` 快速守门，`hotc233ctl perf` 做 PC IL2CPP 三路对比，`hotc233ctl webgl` 做 WebGL 本地 IL2CPP 对比。

商业能力总览

本页参考 HybridCLR 商业版的信息架构，但当前竞品目标只有 HybridCLR Pro 纯解释能力。hotc233 不通过把大量热更逻辑预置进首包 AOT 换性能；生产等价以 WebGL/headless 报告和专项矩阵为准。

完全泛型共享

hotc233-unity 最低支持 Unity 2022+，因此泛型共享只维护 2022+ il2cpp 路线。包内包含 `metadata/GenericSharing.*`、`GenericMetadata.*`、泛型哈希/比较与 Editor AOT 引用分析器；当前属于支持路径，不宣称已达到 HybridCLR Pro 完全泛型共享的生产等价。

Project Settings / Hotc233 Settings

maxGenericReferenceIteration = 10

元数据优化

元数据策略分两层：Editor 侧生成与裁剪 AOT metadata，Runtime 侧通过 `Hotc233LoadPolicy` 对 payload 做访问控制、完整性校验和解密后再调用 `LoadMetadataForAOTAssembly`。

var policy = Hotc233LoadPolicy.AllowOnly("RuntimeMetadata/mscorlib.dll.bytes")
    .RequireSha256("RuntimeMetadata/mscorlib.dll.bytes", expectedHash);

var loader = new HotUpdateBinaryLoader {
    LoadPolicy = policy
};
loader.LoadRuntimeMetadata(metadataBytes, HomologousImageMode.SuperSet);

标准解释优化与离线指令优化

性能测试必须使用 RuntimeFast。当前 PC/WebGL IL2CPP 报告包含机制成本、游戏业务热循环、低 GC 帧循环和内存稳定哨兵；`headless` 与 `validate-reports` 会强制读取原始 JSON，要求 `game-low-gc-frame` 与 `game-memory-stability` 都保持 `gc0=0,gc1=0,gc2=0`，并要求 retained heap delta 不超过 64KB。

HybridCLR Pro 纯解释目标、同机参考表和逐项差距见 HybridCLR 差距说明。

go run ./tools/hotc233ctl headless -project . -loader-profile RuntimeFast
go run ./tools/hotc233ctl perf -project . -loader-profile RuntimeFast
go run ./tools/hotc233ctl webgl -project . -loader-profile RuntimeFast

Hotfix 动态热修复与热重载

CLR/IL2CPP 不会卸载旧 Assembly；hotc233 的 hotfix 语义是更新 loader 解析优先级，并清空类型、方法、delegate 缓存。业务入口应通过 loader 解析或 facade 调用，避免永久持有旧 Assembly 的 MethodInfo/Delegate。

loader.ReplaceHotUpdateAssembly(new NamedBinary(
    "HotUpdateDlls/Feature_HotUpdate.dll.bytes",
    hotfixBytes));

loader.ReloadHotUpdateAssemblies(allHotUpdateDllBytes);

代码加固、加密与访问控制

内置 `Hotc233LoadPolicy` 提供可替换的安全面：开发期可用 XOR 示例，生产环境应接入项目自己的密钥派生、签名服务或平台安全模块。不要把真实生产密钥提交到仓库。

var policy = Hotc233LoadPolicy.CreateXorProtected("dev-key");
policy.EnableAccessControl = true;
policy.AllowedBinaryNames.Add("HotUpdateDlls/Feature_HotUpdate.dll.bytes");
policy.AccessValidator = binary => binary.Bytes.Length < 16 * 1024 * 1024;

loader.LoadPolicy = policy;

Verifyperf/webgl/headless 报告闭环

WebGL/minigame 性能表只统计解释器和 runtime，本地 IL2CPP 对比排除 AssetBundle、YooAsset 和 HTTP 下载链路。

解释器栈崩溃日志

`Hotc233RuntimeDiagnostics` 在每次加载和入口调用时记录 session、环境、payload hash、metadata 加载顺序和最近事件。异常日志会附带 recent-hotc233-events，便于定位 WebGL 真机上不可复现的问题。

HybridCLR 兼容

hotc233 不安装 `com.code-philosophy.hybridclr`，但保留常用 API 和菜单习惯。迁移时可以先把旧打包生态里的入口替换成 `HybridCLR.Hotc233_HybridclrAdapt` 或继续通过 `HybridCLR.RuntimeApi` facade 调用。
