hotc233-unity 固定安装在 Assets/neko233/hotc233-unity，内置生成工具、运行时加载器、元数据补足流程、验证入口和发布自动化。

powershell -NoProfile -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/neko233-com/hotc233-unity/main/install-hotc233-unity.ps1 | iex"
curl -fsSL https://raw.githubusercontent.com/neko233-com/hotc233-unity/main/install-hotc233-unity.sh | sh

工作流

1. 拆分热更程序集

把需要热更的业务代码放进独立 asmdef，主工程通过入口、接口、反射或资源实例化连接热更逻辑。

2. 生成热更产物

运行 hotc233/Generate/All，生成热更 DLL、AOT metadata、bridge、link 保护和资源二进制。

3. 构建前验证

用 hotc233/EditorForBuild/Run Full AB Verification 或项目 CI 确认 DLL bytes 能从资源系统加载并执行。

平台支持口径

平台支持声明和当前仓库验证证据分开记录。发版前以目标平台最新 IL2CPP 验证报告为准。

平台 当前口径 验证要求
Windows IL2CPP 已验证 构建并运行 Player 探针。
Android IL2CPP 已验证 构建验证已通过；发版前补真机安装和启动探针。
iOS IL2CPP 已构建验证 Windows 可验证导出链路；最终需要 macOS/Xcode 真机部署。
WebGL 2 / WebGL IL2CPP 已构建验证 构建验证已通过；最终补浏览器托管 smoke。
Linux IL2CPP 理论支持 需要 Linux module/runner 后纳入矩阵。
macOS IL2CPP 理论支持 需要 macOS runner 后纳入矩阵。
